## What this is

Cyril James De Guzman's personal portfolio/resume site (quinchy.dev), built with **Astro** (`output: "server"`) deployed on **Cloudflare Workers**, with **React islands** for the few interactive pieces and **Tailwind CSS v4**. Bun is the package manager (`bun.lock`).

## Commands

```bash
bun dev           # astro dev (http://localhost:4321)
bun run build     # astro build тАФ prerenders static pages, bundles the Worker
bun run preview   # astro preview (serve the build locally)
bun run deploy    # astro build && wrangler deploy
bun run cf-typegen# wrangler types тАФ regenerate Cloudflare env typings
bun run lint      # biome check (lint + format check + import sort)
bun run format    # biome format --write
```

There is no test suite configured in this repo.

## Architecture

- **Pages** (`src/pages/`): `index.astro` (home тАФ composes all section components), `about.astro`, `projects.astro` (full project list), `404.astro`, and `api/send-email.ts` (POST-only contact endpoint). `robots.txt.ts` is a static endpoint; the sitemap is generated by the official `@astrojs/sitemap` integration.
- **Rendering mode**: `output: "server"` with the `@astrojs/cloudflare` adapter. Content pages opt into static generation with `export const prerender = true`; only `api/send-email.ts` is on-demand (`prerender = false`). **Adding a new content page тЖТ add `prerender = true`** so it's static and gets included in the sitemap.
- **Content is hardcoded in components, not a CMS.** Resume data (projects, experiences, education, skills) lives as typed arrays directly inside `src/components/sections/*.astro`. To update resume content, edit those arrays in place.
- **Draft gating**: project entries can carry `draft: true` (see `src/components/sections/projects.astro`); they're filtered out unless `import.meta.env.DEV`.
- **Layout**: `src/layouts/layout.astro` is the single shell тАФ `<head>`/SEO (canonical, OG/Twitter, JSON-LD `Person`+`WebSite`), the pre-paint theme script, manual scroll restoration, and the section `<slot>`. Filename is lowercase by choice; Astro does not enforce capitalization for layouts/components (the import name is what matters).
- **Theming**: two independent axes тАФ mode (`light`/`dark`/`system`) and color theme (`default`/`teal`/`azure`/`crimson`/`aurora`) тАФ persisted to `localStorage` (`theme`, `color-theme`) and applied as classes (`.dark`, `.teal`, `.azure`, `.crimson`, `.aurora`) on `<html>`. An inline blocking `<script>` in `layout.astro` sets these pre-hydration to avoid a flash; the toggle UI and logic live in `src/components/navbar.astro` as **vanilla JS** (no React provider). CSS variables for each mode/theme combination are in `src/styles/global.css`. When changing theme keys/values, keep the inline script and the navbar script in sync.
- **Components**: shared UI in `src/components/`; reusable primitives in `src/components/ui/` (`badge.astro`, `button.astro`, `button.tsx`, `input.tsx`, `toast.tsx`); page sections in `src/components/sections/*.astro`. **Icons are inline `<svg>` at the call site** тАФ no icon component or library. React is used only where interactivity is genuinely required.
- **React islands**: `contact-form.tsx` and `ui/toast.tsx` are the only hydrated components (`client:load` / `client:only`). Everything else is static Astro/HTML, so the shipped JS is minimal.
- **Contact form flow**: `contact-form.tsx` (react-hook-form + zod) тЖТ `POST /api/send-email` тЖТ re-validates with the same zod schema (`src/validation/contact.ts`), rate-limits via the Cloudflare native `CONTACT_RATE_LIMITER` binding (5 requests/minute per IP), then sends mail with `nodemailer` over Gmail. Email secrets are read from the Cloudflare runtime via `import { env } from "cloudflare:workers"`: `EMAIL_USER`, `EMAIL_PASS` (provide via `.dev.vars` locally and `wrangler secret` in prod). TanStack Query drives the form's mutation (`QueryProvider` wraps the form).
- **Visual flourishes**: CSS film-grain overlay (`body::after` in `global.css` тАФ a `transform`-animated SVG noise, no JS/canvas); section headers via `src/components/section-title.astro`; global toast via an imperative `showToast()` (`ui/toast.tsx`) rather than a React context.
- **Path alias**: `@/*` тЖТ `src/*` (see `tsconfig.json`). Static assets live in `public/` and are referenced by absolute path (e.g. `/projects/<image>.webp`, `/companies/<logo>.webp`).
- **SEO/canonical**: `site` is set in `astro.config.mjs` to `https://www.quinchy.dev` (canonical host); `layout.astro` derives per-page canonical/OG URLs from it.
